# 《深岩银河幸存者》(Deep Rock Galactic: Survivor) 游戏助手 — 简单 PRD

> 文档类型：简单 PRD（确认用，非详细规格）
> 作者：产品经理 许清楚（Xu）
> 状态：已确认，进入架构设计（数据决策：暂不补达成率、空白不高亮；Steam 全球达成率 <10% 视为稀有成就）

## 一、产品目标

**一句话定位**：一个可安装的 PWA 速查 Web 应用，帮《深岩银河幸存者》玩家在游戏中随时检索全成就、武器（含超频模组与强度评级）、装备的关键信息，即开即用、可离线。

**用户价值**：
- 把分散在 Steam 社区攻略、官网、小黑盒的 300 个成就、42 把武器、20 件装备聚合到一个可搜索、可筛选、中英文切换的界面；
- 打游戏时快速确认"这把武器怎么超频/评级如何""这个成就怎么解锁""这件装备该不该拿"，减少切屏查攻略；
- 支持中/英文名与标签检索、强度评级与标签可视化、疑难成就高亮，一眼看清强弱与难点。

**本期范围（已确认）**：P0 + P1 + 中英 UI 切换 + 明暗主题切换（记忆偏好）+ 疑难成就高亮（全球达成率 < 50% 的条目高亮；空白达成率不高亮，不误标）。收藏/对比、导出图片/CSV **本期不做**（见 P2）。

## 二、用户故事（5 条）

1. **作为新手玩家**，我想按职业查看所有武器及其评级（S/A/B/C/-），以便快速决定主玩哪把枪、优先升哪把。
2. **作为冲成就的玩家**，我想按分类筛选成就并看到解锁条件与"疑难高亮"，以便规划"还差哪个成就没拿"。
3. **作为配装玩家**，我想搜索某个标签（如"冰冻""无人机""炮塔"）找到相关武器与装备，以便围绕流派组队。
4. **作为速通/挑战玩家**，我想用关键词（中/英文）模糊搜索（如"爆头""Nitra""护甲"）秒定位对应条目，以便不打断节奏。
5. **作为硬核玩家**，我想在武器卡片上直接看到黄/红超频模组与强度评级，并切换明暗主题/中英文，以便在不同场景下舒适查阅。

## 三、需求池

### P0（必须有，上线即交付）
- **三大模块完整展示**（数据来自上述三个底稿文件，须 100% 录入，不遗漏、不杜撰）
  - 成就：**300 条**，字段 = 英文名 + 中文名 + 分类 + 解锁条件 + 生物群系档位(H5/Mastery/TrueMastery，仅生物群系类填) + 达成率(可空；本期 61 条为空，UI 不高亮) + 版本标记。
  - 武器：**42 把**（44 行含跨职业重复），字段 = 英文名 + 中文名 + 职业(class) + 标签(tags) + 黄色超频(6/12级) + 红色超频(18级) + 强度评级(S/A/B/C/-) + 版本标记。
  - 装备：**20 件**，字段 = 名称 + 类型 + 效果 + 来源(局内附加/成就解锁) + 关联成就(成就解锁类填) + 版本标记。
- **常驻全局搜索**：顶栏搜索框，按中英文名 / 标签 / 条件模糊匹配，跨三大模块即时过滤（输入即筛选，无需点按钮）。
- **分类筛选**（随 Tab 变化）：
  - 成就：按分类筛选（职业解锁/职业进阶/属性统计/装备/武器超频/武器精通/武器真精通/生物群系/武器标签/生物/资源/神器/商店/环境/伤害/护卫/异常·先锋·致命/耐力/其他动作）。
  - 武器：按职业（Scout/Gunner/Engineer/Driller）+ 强度评级（S/A/B/C/-）+ 标签 chips。
  - 装备：按类型 + 来源（局内附加 / 成就解锁）。
- **中英 UI 切换（本期做）**：界面外壳文案中/英切换并记忆；数据本身已含中英双字段。
- **明暗主题切换（本期做）**：明/暗主题切换并记忆偏好（localStorage）。
- **疑难成就高亮（本期做）**：全球达成率 < 50% 的成就在成就模块高亮/加标识（数据见底稿"达成率"列；建议分档：<10% 极难 / 10–30% 难 / 30–50% 较难，避免全屏高亮 — 详见待确认问题）。（注：空白达成率的成就不高亮）

### P1（很重要，确认后优先做）
- **评级可视化**：S 金色徽章、A/B/C 对应色阶徽章、"-"显示"未评级"，避免纯文字难辨。
- **标签 chips**：武器与装备的标签以彩色 chip 呈现，可点击作为快捷筛选条件。
- **响应式布局**：桌面端多列卡片网格，移动端单列堆叠（PWA 主要场景为边玩边查手机/副屏）。
- **数据来源与免责标注**：页脚标注数据来源（Steam 社区攻略 / TrueAchievements / Steam 全球统计）+ 抓取日期 + **"强度评级为攻略作者主观评价，仅供参考"** 的免责说明（必含，因评级非官方）。
- **PWA 可安装**：vite-plugin-pwa 配置 manifest + service worker，支持"添加到主屏幕"、离线访问（数据全内嵌）。

### P2（本期不做，留待后续）
- **收藏 / 对比**：收藏常用武器或成就，武器间并排对比超频与评级。
- **导出**：将筛选结果导出为图片/CSV 分享。
- （中英 UI 切换、明暗主题切换、疑难成就高亮均已列入本期 P0，不再属于 P2。）

### 4.2 各模块卡片字段与控件位置

| 模块 | 卡片字段 | 筛选控件（位于筛选区） | 备注 |
|---|---|---|---|
| 成就 | 中文名、英文名、解锁条件、分类标签、达成率(若有)、⚠疑难徽标 | 分类下拉/19 类 chips、⚠疑难高亮开关 | 生物群系类卡片显示 H5/精通/真精通 档位；达成率<50% 显示疑难徽标 |
| 武器 | 中文名、英文名、职业、标签 chips、黄色超频、红色超频、评级徽章 | 职业下拉、评级下拉、标签 chips | 跨职业武器（电浆卡宾枪、火焰哨戒炮）按职业各显示一条；评级徽章须标注"主观" |
| 装备 | 名称、类型 chip、效果、来源标签、关联成就(若) | 类型下拉、来源下拉（局内/成就解锁） | 成就解锁类卡片显示"关联成就"字段 |

### 4.3 交互要点
- 顶栏（标题 + 语言切换 + 主题切换）与全局搜索、三 Tab 常驻所有页面。
- 搜索与筛选组合作用（AND 关系），输入即更新，无需"查询"按钮。
- 标签 chip 可点击 → 自动填入搜索/筛选（P1，提升检索效率）。
- 主题与语言偏好写入 localStorage，下次访问自动恢复。
- **评级与疑难高亮均须带视觉标识**：评级徽章旁标注"玩家主观评价"；成就疑难以颜色/角标区分达成率分档。
- 卡片采用 MUI `Card` + Tailwind 布局，网格用 `flex-wrap` 实现响应式。

## 五、待确认问题（方案已确认，仅列残留风险）

1. **小黑盒是否成功抓取**：小黑盒（xiaoheihe.cn）成就页存在反爬，本次未能直接抓取结构化数据，已用 Steam 全球统计 + TrueAchievements + Steam 社区攻略兜底（覆盖 1.0 全 300 成就）。若需小黑盒中文译名/攻略补充，待工程期再尝试，或接受当前英文指南译名。
2. **达成率获取完整性**：【已决策】用户确认本期不补全达成率，61 条空白在 UI 不高亮（不误标），遗留至后续迭代补全。当前 239/300 条有全球达成率。
3. **疑难高亮阈值策略**：因 300 成就中约 280+ 条达成率 < 50%，若"全部高亮"会满屏。建议在 UI 用**达成率分档**（极难<10% / 难10–30% / 较难30–50%）做不同视觉权重，而非二元高亮。需主理人/开发确认分档方案。——已确认采用分档方案（极难<10% / 难10–30% / 较难30–50% 三级视觉权重），作为实现细节落地。Steam 官方成就页（https://steamcommunity.com/stats/2321470/achievements）以全球达成率 <10% 界定"稀有成就"，即本分档的"极难"档，UI 须对 <10% 条目施加最强视觉权重（如金色/红色角标）。
4. **武器评级时效性**：评级来自 2025-01 中文攻略（Risk 5 主观评价），1.0 正式版若调整过数值，评级可能需重评；UI 已强制标注"主观评价"，但数据准确性建议上线前复核。
5. **少量武器规范名映射待核**：工程师"等离子切割器"（ArmsKore Coil Gun 或 Breach Cutter）、"等离子手枪"（= Experimental Plasma Charger 为推断）映射待以游戏内文本确认（见 `data-weapons.md` 待核标注）。——工程期保留"待核"标注，不杜撰规范名。
6. **装备列表完整性**：20 件装备基于 2024-11 攻略 + 种子文件，1.0 可能新增/调整局内附加装备，上线前建议以游戏内装备池文本复核补全。——工程期保留版本标注，不杜撰未确认装备。

## 六、给工程师的实现备注（非需求，仅提示）

- 三个底稿文件即应用内数据模块的直接来源（建议拆 `achievements.ts` / `weapons.ts` / `equipment.ts`）。
- 成就"分类"字段已在底稿中统一为 19 类，可直接作为筛选枚举。
